
# TrackerLens

An automated privacy auditing tool designed to unmask invisible third-party tracking domains and background telemetry scripts during conventional web browsing.

## Overview

**TrackerLens** is a lightweight Python-based command-line tool built to analyze web page network requests captured via browser diagnostic archives. Modern websites often execute dozens of third-party scripts that track behavior, serve targeted advertising, and capture user analytical metrics without explicit, transparent user consent. This artifact parses standard HTTP Archive (HAR) logs to identify, categorize, and score network exposure, making invisible web tracking visible to everyday users.

This project was developed as a Capstone artifact for **EGN 101 – Introduction to Computer Engineering** at **Atlantis University**.

## Integrated Academic Concepts

The architecture of this tool directly synthesizes technical principles from three core computational modules:

* **Module 5 – Programming with Python:** Implements clean scripting, automated string-parsing, exception handling, and object-oriented logic to structure multi-layered log processing.


* **Module 6 – Computer Networks:** Inspects underlying HTTP/HTTPS transaction elements, analyzes the client-server request loop, and parses uniform resource identifiers (URIs) from browser network logs.


* **Module 7 – Databases & AI/ML:** Designs local categorical signature schemas to match, filter, and organize raw extracted network logs into logical buckets (Advertising, Analytics, Social Media).

## Step-by-Step Execution Guide

To run a privacy audit on your favorite websites, follow these exact steps to feed a network archive into the script:

### Step 1: Export a Network Archive (.har) from Your Browser

1. Open your preferred web browser (e.g., Google Chrome, Mozilla Firefox, or Microsoft Edge).
2. Navigate to the target website you wish to evaluate (e.g., a major news outlet or an e-commerce hub).
3. Right-click anywhere on the page and select **Inspect** (or press `F12`) to open the **Developer Tools**.
4. Switch directly to the **Network** tab at the top of the interface.
5. Refresh the page (`F5`) to allow the browser to capture all background telemetry, image loads, and API requests.
6. Once the page completes loading, click the **Export HAR...** download icon (a down-facing arrow) or right-click anywhere inside the transaction log list and select **"Save all as HAR with content"**.

### Step 2: Prepare the Workspace

Move your newly exported file into the root folder where `tracker_analyzer.py` is saved, and make sure it is named exactly:

```text
example.har

```

### Step 3: Execute the Script

Run the application from your terminal workspace:

```bash
python tracker_analyzer.py

```

## Ethical Guidelines Applied

This tool operates under strict compliance with professional ethical frameworks outlined by computing institutions:

* **ACM 1.6 (Respect Privacy):** Engineered to increase web transaction transparency by shedding light on unconsented background data collection frameworks without generating harmful tracking profiles of its own.
* **ACM 1.2 (Avoid Harm):** Empowers daily internet users to audit and recognize malicious, intrusive, or manipulative data-harvesting trackers.
* **IEEE I.2 (Improve Understanding of Technology):** Desublimates hidden browser operations into clear, scannable data summaries, advancing technical awareness for non-engineering stakeholders.
